# Contributing to Walden / Rocinante

If you're even thinking about contributing, thanks!

## Reporting Issues

Please report bugs to the [GitHub issue tracker](https://github.com/rail5/walden/issues). Feature requests and design discussions are also welcome there.

Also, "I have a hard time understanding this part of the code" is a **very** valid issue to report. One of the major goals here is to make the codebase as approachable/accessible/easy to understand as possible -- so as far as I'm concerned, "unreadable code" is a bug.

If you'd prefer to report issues privately, you can also email me directly.

## Contributing Code

Because this is a very low-level project (a kernel + operating system), the standards for code quality are quite high. "Slipping up" costs a lot more in a kernel than in some web backend.

Please see [STANDARDS.md](STANDARDS.md) for details on code quality standards and documentation expectations.

That doesn't mean you shouldn't contribute if you're not an expert -- in fact, I hope very much that people who are new to low-level programming will contribute! Please do however put in the effort to meet these standards. If you're not sure about something, you can ask!

Once you have code to contribute, please open a Pull Request on GitHub, or email me patches directly if you prefer.

### Licensing

This project is GPL-3.0+ licensed, and will not accept any contributions without *explicit* agreements to license under GPL-3.0+.

That means that you must *explicitly* agree to license *your* code under GPL-3.0+ in order to contribute it to this project. This is a legal requirement, not just a formality.

The GitHub Pull Request template includes this agreement already, but if you're emailing patches, please include a statement to that effect in your email.

## Code of Conduct

Our code of conduct is very simple: "don't be a jerk." See the [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) file if you must, but it's fairly self-explanatory.
